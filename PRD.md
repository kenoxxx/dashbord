🚀 PRD: Prompt Nebula (Локальный дашборд промпт-инженера)
1. Обзор продукта (Product Overview)
Суть: Локальное интерактивное веб-приложение для управления библиотекой атомарных промптов, генерации текстов по шаблонам с динамическими переменными, а также безопасного хранения заметок и паролей к AI-сервисам.
Платформа: Локальный сервер (Python + Streamlit).
Стиль: Космический минимализм (темные глубокие тона, неоновые акценты, отсутствие визуального мусора).

2. Технический стек и Архитектура
Фреймворк: Python + Streamlit.

Хранилище данных: Локальные JSON файлы (выступают в роли базы данных).

prompts.json — для промптов и категорий.

vault.json — для паролей и заметок.

Форматирование: Все тексты промптов сохраняются и рендерятся с поддержкой Markdown (.md). Будет добавлена кнопка "Экспорт в .md файл" для скачивания.

Дизайн: Кастомный CSS внутри Streamlit (темный фон #0b0f19, акценты цвета звездной пыли/неона #00f0ff или #b026ff, скругленные формы, моноширинные шрифты для кода).

3. Модель данных (Data Structure)
База будет разделена на три логических блока:

Категории: Динамический список. Дефолтные: "AI РАЗРАБОТКА", "Генерация текстов, постов, контент планов". Пользователь может добавлять новые.

Промпты:

ID, Название, Категория, Шаблон (текст в Markdown)

Vault (Хранилище паролей/заметок):

Сервис, Логин, Пароль, Заметки

4. Ключевой функционал (Core Features)
🔹 Блок 1: Управление промптами (Prompt Library)
Просмотр и фильтрация: Выбор категории из выпадающего списка.

Создание промпта: Форма с вводом названия, выбором/созданием категории и большим полем ввода для Markdown-текста.

Копирование и Экспорт: Кнопка "Скопировать" и кнопка "Сохранить как .md файл".

🔹 Блок 2: Динамические шаблоны (Dynamic Templating)
Умный парсинг: Система ищет в тексте промпта переменные в квадратных скобках [...] (например, [роль: эксперт], [тип контента]).

Генератор интерфейса: Для каждой найденной переменной дашборд автоматически создает текстовое поле (Input).

Сборка: При заполнении полей генерируется итоговый текст промпта, готовый к копированию. Пользователь может вставить "Золотой промпт", заполнить нужные поля (Аудитория, Боль, Формула) и скопировать готовый результат одним кликом.

🔹 Блок 3: Секретное хранилище (Vault / Notes)
Отдельная вкладка или изолированная зона в интерфейсе.

Таблица или карточки с учетными данными для ChatGPT, Midjourney, Claude, локальных LLM и т.д.

Скрытые пароли (показываются по кнопке-глазику или просто копируются в буфер обмена).

5. Пользовательский интерфейс (UX/UI)
Интерфейс будет состоять из бокового меню (Sidebar) и основной рабочей зоны.

Боковое меню (Навигация):

Переключатель разделов: 🌌 Библиотека Промптов | 🛠 Конструктор (Шаблоны) | 🔐 Vault (Пароли)

Форма быстрого добавления новой категории.

Основная зона (Библиотека/Конструктор):

Строка поиска и фильтр по категориям.

Карточки промптов в виде аккордеонов (раскрывающихся списков).

При раскрытии промпта-шаблона появляются поля для ввода переменных и блок с финальным результатом.

6. План разработки (Roadmap)
Этап 1 (База): Настройка Streamlit, кастомный CSS (космический стиль), создание структуры JSON файлов.

Этап 2 (CRUD промптов): Добавление, удаление, просмотр промптов. Работа с категориями.

Этап 3 (Движок шаблонов): Написание регулярного выражения (Regex) для поиска [...] и генерации полей ввода. Сборка финального текста.

Этап 4 (Vault): Создание раздела для паролей и заметок.

Этап 5 (Полировка): Добавление кнопок копирования, экспорта в MD, проверка работоспособности "Золотого промпта".
